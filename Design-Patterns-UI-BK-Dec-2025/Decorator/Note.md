## Decorator Design Pattern

### Category: Structural

### Purpose: Add extra behavior or features to an existing object without modifying its original structure or class.

### Best use-case: When you want flexible, composable enhancements such as:
```
    Logging / caching / validation

    Styling or formatting

    Conditional behaviors

    Feature toggles

    Optional UI capabilities
```

### Frontend use-case: Enhancing a UI Button component by decorating it with: styles, icon, hover-effects

### Backend use-case: Tea Maker with varierty of ingredients & Price Calculation
